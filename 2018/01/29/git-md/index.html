Git Cheat Sheet

发表于 2018-01-29 |

廖雪峰Git教程

git安装：

$ brew install git

git配置：
创建仓库：

pwd 用于显示当前目录

初始化git仓库：

$ git init

添加文件到git仓库：
第一步:

$ git add file.txt

或者添加多个文件

$ git add file1.txt file2.txt

第二步：

$ git commit -m "Your Comments"

查看仓库当前状态：

$ git status

查看具体的修改内容：

$ git diff file.txt

$ git diff HEAD -- file可以查看工作区和已提交到版本库的文件内容差异

查看当前窗口下提交commit的历史记录：

$ git log

添加参数可使显示信息简洁：

$ git log --pretty=oneline

或

$ git log --pretty=oneline --abbrev-commit

将文件回溯到上一个版本或更久的版本：

$ git reset --hard HEAD^

HEAD 表示当前版本，上一个版本就是HEAD^，上上个版本就是HEAD^^，上100个版本可以写成HEAD~100

在当前命令行窗口未关闭的前提下，提供 commit id 仍然可以回到未来的某个版本：

$ git reset --hard Your Commit id

查看命令历史：

$ git reflog

当改乱了工作区某个文件的内容，想直接丢弃工作区的修改时，用命令：

$ git checkout -- file

不但改乱了工作区某个文件的内容，还添加到了暂存区时，想丢弃修改，分两步，第一步用命令：

$ git reset HEAD file
把暂存区的修改撤销掉（unstage），重新放回工作区

第二步再用：

$ git checkout -- file

已经提交了不合适的修改到版本库时，想要撤销本次提交，前提是还没推送到远程库：

参考上面 $ git reset --hard HEAD^ 部分

删除文件操作：

直接删除工作区文件可直接在shell利用命令：

$ rm file

如果一个文件已经被提交到版本库，发现工作区文件误删，可利用命令恢复至版本库里的最新版本：

$ git checkout -- file

如果是确定连版本库的文件也删除，更彻底方式是：

创建SSH Key:

$ ssh-keygen -t rsa -C "youremail@example.com"

关联远程库：

$ git remote add origin Your HTTP or SSH

远程库的名字就是 origin ，这是Git默认的叫法，也可以改成别的。

关联后第一次推送master分支的所有内容：

$ git push -u origin master

参数 -u 是指Git不但会把本地的 master 分支内容推送到远程新的 master 分支，还会把本地的 master 分支和远程的 master 分支关联起来，在以后的推送或者拉取时就可以简化命令。

此后，每次本地提交后，只要有必要，就可以使用命令

$ git push origin master 推送最新修改

克隆仓库（推荐使用SSH协议，速度快，省去口令）：

$ git clone Your Repo Address

创建dev分支，并切换到dev分支：

$ git checkout -b <name>

其中 get checkout 命令加上参数 -b 表示创建并切换，相当于两条命令：

查看当前分支：

$ git branch

切换分支：

$ git checkout <name>

合并某分支到当前分支：

$ git merge <name>

删除分支：

$ git branch -d <name>

查看分支：

$ git branch

查看分支合并图：

$ git log --graph

查看分支合并的具体commit信息并以图的形式给出：

$ git log --graph --pretty=oneline --abbrev-commit

通常，Git会用Fast forward 模式，但这种模式下，删除分支后，会丢掉分支信息，看不出曾经做过合并。

合并分支时，加上 --no-ff 参数就可以用普通模式合并，合并后的历史有分支，能看出来曾经做过合并

$ git merge --no-ff -m "Your Comments" <name>

加上 -m 参数，把commit描述写进去

当手头工作未完成，可以先把工作现场储存，等以后恢复现场再继续工作：

$ git stash

恢复工作前先用命令 $ git stash list 查看要恢复的stash

恢复时有两种方式

命令一：

$ git stash apply stash@{stashnumber}

恢复后，stash内容并不删除，你需要用 git stash drop 来删除

命令二：

$ git stash pop stash@{stashnumber}

恢复的同时把stash内容也删了

如果要丢弃一个没有被合并过的分支，需要强行删除：

$ git branch -D <name>

开发一个新feature，最好新建一个分支

查看远程库信息:

$ git remote -v

从本地推送分支:

$ git push origin branch-name

如果推送失败，先用$ git pull 抓取远程的新提交

在本地创建和远程分支对应的分支:

$ git checkout -b branch-name origin/branch-name

本地和远程分支的名称最好一致

建立本地分支和远程分支的关联:

$ git branch --set-upstream branch-name origin/branch-name

从远程抓取分支:

$ git pull

如果有冲突，要先处理冲突

多人协作模式：
  • 首先，可以试图用 $ git push origin branch-name 推送自己的修改；
  • 如果推送失败，则因为远程分支比你的本地更新，需要先用 $ git pull 试图合并；
  • 如果合并有冲突，则解决冲突，并在本地提交；
  • 没有冲突或者解决掉冲突后，再用 $ git push origin branch-name 推送就能成功！
  • 如果 $ git pull 提示“no tracking information”，则说明本地分支和远程分支的链接关系没有创建，用命令
    $ git branch --set-upstream branch-name origin/branch-name
打标签：
  • 首先切换到要打标签的分支上；
  • 命令 $ git tag <name> 用于新建一个标签，默认为 HEAD ，也可以指定一个commit id，$ git tag <name> Commit ID

$ git tag -a <tagname> -m "blablabla..." 或后面加CommitID， 可以指定标签信息；

$ git tag -s <tagname> -m "blablabla..." 或后面加CommitID， 可以用PGP签名标签；

$ git tag 可以查看所有标签

$ git show <tagname> 查看标签信息

推送某个标签到远程：

$ git push origin <tagname>

一次性推送全部尚未推送到远程的本地标签:

$ git push origin --tags

删除一个本地标签：

$ git tag -d <tagname>

如果标签已经推送到远程，要删除远程标签分两步：
  • 先从本地删除$ git tag -d <tagname>
  • 再从远程删除 $ git push origin :refs/tags/<tagname>
忽略某些文件时，需要编写:

.gitignore

.gitignore 文件本身要放到版本库里，并且可以对.gitignore做版本管理！

若此时想添加一个文件到Git，但发现添加不了，原因是这个文件被.gitignore忽略了;

如果你确实想添加该文件，可以用-f强制添加到Git：

$ git add -f filename

或是想找到.gitignore中哪个规则写错了：

$ git check-ignore -v filename
